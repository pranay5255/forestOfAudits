EVMBench Modal Forest

Capture every agent and tree trajectory without losing partial runs.

The forest runner already gives each scout, branch, tree judge, and global judge a separate mini-swe-agent trajectory path. The practical fix is to run with a fixed output directory, keep partial workers with --continue-on-worker-error, and verify the filesystem with find, not just metadata.

1 scout trajectory at logs/forest/scout.traj.json
N x B branch trajectories under logs/forest/<role>/
N tree judge trajectories when branches reach judge stage
1 global judge trajectory when synthesis reaches final stage
1. Mental model

Where the data is written

Mini-swe-agent writes trajectory JSON from the host-side runner process. Remote worker artifacts such as branch reports are extracted from Modal sandboxes, but the trajectory files live directly under the local forest output directory.

01

Scout

Maps the audit surface and selects roles unless --tree-roles is supplied.

logs/forest/scout.traj.json

02

Branch workers

Each role and branch gets its own isolated worker and trajectory file.

logs/forest/<role>/branch-01.traj.json

03

Tree judges

Each selected role gets a judge that reads branch reports and writes a tree report.

logs/forest/<role>/judge.traj.json

04

Global judge

Reads tree reports and writes the final detect submission.

logs/forest/global-judge.traj.json

05

Summary

Metadata records selected roles, worker runtimes, errors, and trajectory paths.

logs/modal-forest-result.json

Important capture rule

modal-forest-result.json is useful, but it may not list workers from a batch that crashed before orchestration collected the future. Always use find "$OUT" -type f -name '*.traj.json' as the source of truth for files that were actually written.

5. Troubleshoot

What missing files usually mean

A missing trajectory is usually a stage reachability issue, not a copy issue. The path exists only after that worker starts.

Only scout.traj.json exists
The run reached scout but did not successfully launch branch workers. Check logs/modal-forest-result.json, logs/modal-runner.stderr.log if running through Phase 6, and the final message in the scout trajectory.
Branch trajectories exist, but no judge.traj.json
Branch stage failed before tree judges launched, or worker errors stopped the run. Add --continue-on-worker-error for debugging and keep --worker-concurrency 1 until capture is reliable.
Metadata lists fewer workers than find
This can happen when a batch crashes before all futures are collected. Prefer filesystem discovery for capture, then use metadata as the annotation layer.
No trajectories appear in a Phase 6 run
Make sure you are searching below the individual task run directory, not only the Phase 6 root. The production path is <phase6-root>/<runner>/<run-group>/<audit_uuid>/modal/logs/forest/.
